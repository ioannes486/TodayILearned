## 10.2 객체의 타입변환
객체의 타입변환에는 두가지 종류가 있다.
`업캐스팅` : 자식 클래스에서 부모클래스로 가는 것
`다운캐스팅` : 부모클래스에서 자식 클래스로 가는 것

캐스팅은 다음과 같은 법칙을 따른다
1. 업캐스팅은 무조건 가능하다.
2. 다운캐스팅은 조건부로 가능하다.
```java
// A<-B<-C일 때
Parent a1 = new Child(();
A a1 = new B(); // 가능
					  
B b1 = (B)a1	// A부터 B까지 다운캐스팅해서 선언했으므로 이런 식은 가능	
//후   전
C c1 = (C)a1  // 하지만 C는 중간에 없으므로 이런 식은 불가
```
위와 같이 선언을 할 때 더 낮은 단계의 자식클래스일 수록 더 많은 변환이 가능하다.

3. 캐스팅의 가능 여부는 무슨 타입으로 선언돼 있는지는 중요하지 않으며 어떤 생성자로 생성됐는지가 중요하다.
4. 다운 캐스팅을 할 수 있기 위해서는 힙 메모리 내에 해당 객체가 있어야 한다.

## 10.4 인스턴스 필드와 정적멤버의 중복

> 인스턴스 필드나 static 필드와 static메서드는 자식클래스에서 동일한 이름으로 정의해도 오버로딩되지 않는다.

### 10.4.1 인스턴스 필드의 중복

인스턴스 필드는 상속받을 필드와 동일한 이륾으로 자식 클래스에서 정의해도 각각의 저장공간에 저장되므로 오버라이딩은 발생하지 않는다.

### 10.4.2 정적 필드의 중복

정적필드명을 중복해 상속해도 저장공간이 분뢰되어 있으므로 오버라이딩은 발생하지 않는다.
